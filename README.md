**Likely** is a real-time, AI-powered party game where friends vote on "most likely to..." questions. The project's primary goal is to serve as a portfolio piece demonstrating a professional-grade, distributed systems architecture.

---

## What is Likely?

Likely is a social game designed for mobile browsers. A host creates a room, chooses a question pack, and invites friends with a simple code. Players vote on who they think is "most likely to" do something outrageous. The twist? The game features a **Live Persona Sync**, where an AI analyzes player votes in real-time to build evolving personas and generates a hyper-personalized final question based on the unique dynamics of the lobby.

### Key Features

* **Real-Time Gameplay:** Lobbies, chat, and voting results are all updated in real-time using WebSockets.
* **OTP Authentication:** Fast and secure user registration and login.
* **AI-Generated Content:** Question packs are generated by AI, and the final question is hyper-personalized to the players in the lobby.
* **Scheduled Games:** Users can schedule a daily question for their room, turning the game into a fun routine.
* **Live Chat:** Every room has a chat that resets after each question, keeping the conversation fresh.

## System Architecture

This project is not a monolith. It is a **distributed, event-driven system** designed for scalability, resilience, and maintainability. The frontend and backend are completely decoupled and communicate via a contract of HTTP and WebSocket events.

### Core Components

1.  **Frontend Client (SvelteKit on Vercel):** A pure presentation layer. Its only jobs are to render the UI based on state received from the server and maintain a persistent, authenticated WebSocket connection.

2.  **Backend Services (Go on Fly.io):** A single Go binary logically structured into decoupled services that communicate internally via channels (simulating a message bus like NATS).
    * **Auth Service:** Manages user identity via OTP and issues JWTs.
    * **Real-time Core:** Manages the lifecycle of all authenticated WebSocket connections.
    * **Game Logic Service:** A state machine that controls the flow of each game.
    * **AI Persona Service:** Interfaces with the OpenAI API for question generation and live persona analysis.
    * **Scheduler Service:** A cron-based job scheduler for daily questions.

3.  **Database (PostgreSQL on Fly.io):** The single source of truth for all persistent data, including users, lobbies, and questions.
